wireframes explaination:

When we look at my new and updated wireframes as portrayed in the images above, I really love how they turned out along with how my current website looks very similar to my latest designs with a few changes made here and there, as you can see my website is not identical to my wireframe. In this I have decided to discuss each section and any major changes I made.

Home page: from the initially page I removed the ‘Harper’ text displayed and implemented a moving text that displays who I am which you can see as it changes. In conjunction I utilised only one font there instead of two as I felt more drawn to the ‘Lara’ font and felt that the ‘Brown Sugar’ was overbearing and did not bring out my minimalism website idea the way I wanted it to.


About page: This page came out exactly how I wanted it, although I made a few changes with the placement of my first names and surname. The only thing added to this was a scroll up feature for when the user would like to go to the top but doesn’t need to scroll.


Portfolio page: For this section, it mainly relied on the images I intended to use for it, these images being my model images from when I work with makeup artist, designers etc. So with this there are three various sections being editorial, beauty and studio. Each page opens into a new page displaying the images related to the portfolio. I made sure that these images were less than 1MB so that they can load in time and the user does not struggle waiting for the images because of their size.


Contact page: I really struggled trying to centre the send message text on the button especially when the screen has been made responsive for the various devices along with trying to centre my icons and text for the responsive pages.


Dev Blog page: For the devblogs I made a normal page section wherein you can select which devblog you would like to go into, instead of having all of them displayed on one page which forces the reader to go through them all instead of picking which one they would like to see. This page also made use of a scroll up feature for when the user would like to go to the top but doesn’t need to scroll and is available for each devblog.


Essay Page: This page has been structure in the same way the devblog page has, however the only difference here is that the essays are two whereas the devblogs are thirteen.


Design Page: For this page I kept it simple as you can see in the wireframe images, however adding my previous design notes along with my new notes and some images if needed to display these notes.

Font: The fonts I used was only one which is ‘Lara’ as I decided that there was not much more of a function for the previous second font ‘Brown Sugar’ as I changed the home page of my website to accommodate the new JS element implemented, while trying to stick to the minimal idea.


Colour Scheme: The colour scheme I decide to use for my website were white and the various shades of black and grey.

documentation of ixd:

Goal Alignment: The goal of my website is to create a website portfolio for my modelling work that showcases various kinds of work I’ve done which will help me receive more possible opportunities to work with other photographers, makeup artists, designers, models, etc. With this I would also like to expand on my website by including more work I’ve done which such as runway (as I could not include it as I have only walked for three designers) I wanted people to feel free to reach out to me so that we are able to collaborate and create content, therefore growing both our portfolios whether they are a photographer or makeup artist, etc.


Interface elements: My website can be seen to make use of the following interface elements, the first one being the navigation bar, which on the home page appear on top of the screen, however when the screen is responsive it will appear as a collapsed nav bar depending on the size of the screen. how does ui and ux impact your code:

When it comes to the user interface (UI) and user experience (UX), the design of your website can impact your code structure and organisation. Hence why it is important to create folders that reflect the hierarchical information to establish a clear understanding of your code and what each part functions. Hence why I created a new website, using the same HTML, CSS and JS from the previous assignments and made a few changes where needed, especially by removing unnecessary code or stuff that does not align or make sense.


I decided to leave the main pages of the website outside folders and any subpages or styling element into folders to make the information look clear and understandable. With this mentioned it need to separate HTML, JS, and CSS on a logical sense. This was established through in the beginning of the paragraph. This separation improves code structure, maintenance, and reuse. HTML files, for example, should largely contain the structure and content of web pages. Another thing to note is that removing most of the JS I had in my previous websites HTML code is essential so that the person accessing my code can check my JS out along with having my website show that it does contain JS


Naming conventions are essential for attaining modularity and scalability. For having the correct conventions allow the code to be easily read and seen to match the content that the code is functioning to display.
